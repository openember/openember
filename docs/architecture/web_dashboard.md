# web_dashboard 设计说明

本文档描述 `apps/services/web_dashboard` 的页面信息架构、职责边界与后续扩展方向。实现上采用 **Layui + 静态 `web_root`**，不依赖前端构建链，便于嵌入式与离线部署。

## 1. 目标与约束

- **目标**：为基于 OpenEmber 的设备（如机器人、边缘网关）提供浏览器内的 **状态总览、控制入口、参数配置、日志查看、升级与合规信息**。
- **约束**：默认无 Node/Vite；静态资源可放 `web_root` 或通过 Kconfig 指定目录；HTTP 服务由 Mongoose 在 `main.cpp` 中提供，`/api/*` 可逐步补齐。

## 2. 信息架构（侧栏）

| 路由键 (`data-page`) | 中文名 | 职责概要 |
|---------------------|--------|----------|
| `overview` | 概览 | **Dashboard**：系统/硬件摘要（类 neofetch）、模块状态占位、运行日志示例。对应首页。 |
| `control` | 控制 | **可操作能力**：数字/模拟开关、摇杆、视频/图传预览、点云、IMU/三维位姿等；与机器人业务强相关，多通过后续 `/api/control/*` 或消息总线桥接。 |
| `settings` | 设置 | **设备参数**：设备名、Wi‑Fi、IP、主机名等；类似路由器管理页；后续对接配置持久化与网络栈。 |
| `logs` | 日志 | **集中日志**：展示汇聚到 web_dashboard 的日志流；支持等级筛选、检索、导出/下载。 |
| `update` | 更新 | **OTA**：OpenEmber 应用版本、系统/固件升级流程、进度与回滚策略占位。 |
| `about` | 关于 | **项目与合规**：OpenEmber 简介、用户条款链接、开源 **SBOM** 清单等。 |

侧栏与内容区通过同一文档内 **单页切换**（`display` 显隐）实现，避免多 HTML 文件与重复壳布局；后续若页面过大可拆分为独立 HTML 并由服务端路由。

## 3. 各页功能细化（当前以布局与占位为主）

### 3.1 概览（overview）

- 展示从 `/api/system` 拉取的 OS、Host、Kernel、Uptime、CPU、GPU、Memory 等（已实现部分）。
- 模块列表、进度条、示例日志可为 mock，后续接真实 `/api/modules`、`/api/metrics`。
- 作为默认落地页；可仅在概览页活跃时轮询系统信息以降低负载（实现可选）。

### 3.2 控制（control）

- **开关**：继电器/灯光/使能等数字量（占位按钮 + 状态灯）。
- **摇杆**：二维摇杆 UI 占位（后续可映射到速度指令或云台）。
- **图传**：`<video>` 或占位画布（WebRTC / MJPEG / HLS 由设备能力决定）。
- **点云**：WebGL/Three.js 为可选增强；初版可用「占位卡片 + 说明」。
- **位姿/IMU**：数值只读区 + 简易 3D 占位（或外链到专用工具）。

后端契约建议预留：`POST /api/control/...` 或经 msgbus 的 RPC，本文档不锁定具体路径。

### 3.3 设置（settings）

- 分组：**设备身份**（名称、序列号）、**网络**（主机名、IP/掩码/网关、DNS、Wi‑Fi SSID 等）、**高级**（预留）。
- 表单校验与「应用/重启」流程后续实现；当前可为只读或本地假保存提示。

### 3.4 日志（logs）

- 列表或终端风格视图；筛选：**等级**（DEBUG/INFO/WARN/ERROR）、关键词、时间范围。
- **下载**：导出当前筛选结果为文本/JSON；可由后端生成临时文件或前端拼接 Blob。
- 数据来源：SSE/WebSocket、`GET /api/logs?...` 轮询、或 msgbus 订阅后经聚合服务写入。

### 3.5 更新（update）

- 展示 **当前软件版本**（构建 ID、Git SHA 可由 CMake 注入）、**通道**（stable/beta）。
- **固件/系统镜像**升级分区与 **应用包**升级分区可分开说明。
- 进度条、校验和、失败重试与「上次成功版本」占位。

### 3.6 关于（about）

- 项目简介与文档链接。
- **用户条款 / 隐私**：外链或静态页。
- **SBOM**：表格列出主要依赖（Mongoose、Layui 等）及许可证；可与仓库 `third_party` 或生成脚本同步。

## 4. 前端技术选型

- **Layui 2.x**：布局、表格、表单、按钮、进度条；CDN 或离线 `web_root/layui`。
- **主题**：与 OpenEmber 品牌一致的橙色主色（CSS 变量覆盖）。
- **数据获取**：`fetch` 为主，`XMLHttpRequest` 作兜底；核心逻辑不依赖 `layui.use` 成功，避免 CDN 不可用导致整页无请求。

## 5. 后端与配置（摘要）

- 静态根目录、HTTP 端口：见根 `Kconfig` / `OPENEMBER_WEB_DASHBOARD_*` 与 `main.cpp`。
- API 逐步从静态占位演进为真实设备数据；建议统一 JSON 与错误码约定（可与 OSAL/组件层错误码对齐）。

## 6. 后续工作（非本次范围）

- 各页面对应 REST/SSE 契约与鉴权（Session、Token 或设备本地密码）。
- control 页实时性与安全（速率限制、急停）。
- logs 大流量下的环形缓冲与分页。
- OTA 与真实引导程序/包管理器的集成。
