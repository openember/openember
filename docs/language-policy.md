# 语言路线政策（C++ 主线 + C ABI）

## 目标

在保证嵌入式/传统生态可接入的前提下，避免维护“双主线同构实现”的指数级复杂度。

## 决策

1. **C++ 为主线**：核心 runtime、调度、插件、配置、观测等能力优先在 C++ 内实现并演进。
2. **C 提供边界兼容**：保留稳定的 **C ABI**（opaque handle + 函数门面），为纯 C/ MISRA 受限场景提供“可用子集”接入能力。
3. **能力分级**：
   - L1（建议公开）：pub/sub 基础、配置读写、日志、健康状态等。
   - L2（通常仅 C++）：插件热加载、复杂异步编排、高级观测等。

## C 接入模板的 transport 约束

当用户启用 “C 接入模板” 时，通信骨架/transport 只能选择 **C 可直接链接**的实现（例如 NNG/LCM/libzmq 的 C 侧接口）。
